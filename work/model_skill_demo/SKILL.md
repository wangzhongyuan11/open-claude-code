# Demo Review Skill

## When to use
- Review demo code for quality and functionality
- Analyze implementation details of demo projects
- Provide constructive feedback on code structure
- Evaluate adherence to best practices for demo applications

## Workflow
1. **Initial Analysis** - Quickly scan the project structure and identify key files
2. **Code Quality Check** - Evaluate code structure, readability, and maintainability
3. **Functionality Review** - Test core features and verify they work as intended
4. **Best Practices Assessment** - Check for adherence to coding standards and guidelines
5. **Feedback Compilation** - Summarize findings and provide actionable recommendations

## Key Focus Areas
### Code Quality
- Readability and maintainability
- Consistent coding style
- Proper documentation
- Error handling

### Functionality
- Core features implementation
- Edge case handling
- Performance considerations
- User experience

### Best Practices
- Design patterns
- Security considerations
- Testing approach
- Version control practices

## Tools and Techniques
- Static code analysis
- Manual code review
- Unit test evaluation
- Integration test assessment

## Output Format
- Summary of strengths and weaknesses
- Detailed feedback on specific issues
- Actionable recommendations for improvements
- Code examples for suggested changes

## Quality Expectations
- Deliver clear and actionable feedback
- Focus on both technical and user experience aspects
- Maintain a constructive and helpful tone
- Provide specific examples and suggestions for improvement
